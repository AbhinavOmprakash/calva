Paredit is active when you edit code in this window. Check and adapt the default Paredit keyboard shurtucts in VS Code's shortcuts pane. Some other useful shortcuts (defaults) are:

  • alt+up/down: for history navigation
  • alt+enter: to submit the current line
In a code editor these shortcuts (defaults) are extra handy:
  • alt+ctrl+c alt+e: to evaluate the current form in this window
  • alt+ctrl+c alt+space: to evaluate the current top-level form in this window (also works within (comment) forms
  • alt+ctrl+c alt+n: to make his window's current namespace the same as the editor's
Note that when evaluating forms from an editor, this window will always use the editor's namespace to evaluate it. Also please note that often you don't need to send forms here to examine their results. There is also editor inline evaluation:
  • alt+ctrl+c e: inline evaluate the current form
  • alt+ctrl+c space: inline evaluate the current form
(The general pattern is that the inline evaluation shortcuts skip the `alt` modifier.)

It is a good habit to start any Calva session by evaluating some files. Many things do not work before that is done. Use ctrl+alt+c enter to evaluate the active file.
